# 📦 Array-Based Linked List with Free List Reutilization

이 프로젝트는 동적 할당(`malloc`/`calloc`)의 오버헤드를 줄이기 위해 **물리적인 배열 공간 위에서 논리적인 연결 리스트를 구현한 자료구조**입니다.

특히 구조체 내부에서 사용 중인 노드를 추적하는 **Active List** 체인과, 삭제된 공간을 관리하는 **Free List** 체인을 별도의 추가 메모리 없이 단일 `next` 필드 하나만을 공유 및 돌려막기(Multiplexing)하여 메모리 효율성을 극대화한 것이 핵심 설계 특징입니다.

---

## 🛠️ 핵심 메커니즘 (Key Concepts)

### 1. 단일 포인터(`next`)의 이중 역할화 (Field Multiplexing)

기존의 배열 기반 연결 리스트는 사용 중인 노드를 위한 `next` 필드와 삭제된 가용 공간을 엮기 위한 별도의 `Dnext` 필드를 따로 두는 경우가 많습니다. 하지만 본 구현체는 노드의 논리적 상태에 따라 `next` 변수를 완전히 다른 의미로 전환하여 메모리 다이어트를 달성했습니다.

* **Active Node (사용 중)**: 일반적인 연결 리스트의 Pointer 역할을 수행하며, **다음 데이터 노드의 물리 인덱스**를 저장합니다.
* **Free Node (삭제됨)**: 가용 공간을 관리하는 Stack 구조의 Pointer 역할을 수행하며, **다음에 재사용할 수 있는 빈 인덱스**를 저장합니다.

### 2. Stack 구조 기반의 자원 회수 및 LIFO 재활용

* **자원 회수 (`delete_index`)**: 노드가 삭제되면 시스템에 메모리를 반환하는 대신, 해당 인덱스를 `list->deleted` 체인의 맨 앞(Top)에 Push합니다.
* **자원 재활용 (`get_index`)**: 새로운 노드가 삽입될 때, 항상 `list->deleted`를 먼저 확인합니다. 만약 회수된 빈 인덱스가 존재하면 배열의 새로운 공간을 파먹지 않고 최신에 해제된 가용 공간을 먼저 꺼내어(Pop) 완벽하게 재사용합니다.

* Active List 순회: head 변수에서 출발하여 각 노드의 next 필드가 양수(0 이상)인 동안 체인을 따라갑니다. 끝은 null (-1)입니다.
