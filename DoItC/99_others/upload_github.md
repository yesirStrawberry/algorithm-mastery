## GitHub에서 프로젝트를 다운로드(Clone)받아 수정하고 다시 올리는(Push) 전체 과정

이 흐름은 개발할 때 매일같이 사용하는 가장 표준적인 Git 워크플로우입니다.

---

### 🔄 Git 전체 흐름 한눈에 보기

1. **GitHub (원격 저장소)** ➡️ `git clone` ➡️ **내 컴퓨터 (로컬 저장소)**
2. **코드 수정 및 파일 변경** (작업 공간)
3. **변경 사항 장바구니 담기** ➡️ `git add .` ➡️ **스테이징 영역**
4. **버전 기록(영구 저장)** ➡️ `git commit -m "메시지"` ➡️ **로컬 저장소**
5. **인터넷에 최종 업로드** ➡️ `git push` ➡️ **GitHub (원격 저장소)**

---

### 📝 실전 Git 활용 치트시트 (그대로 따라 하기)

#### 1단계: GitHub에서 내 컴퓨터로 프로젝트 다운로드 (최초 1회만)

원하는 폴더(예: `DoItC`)로 이동한 뒤, GitHub 저장소 주소를 복사해 다운로드합니다.

```bash
# 1. 다운로드받을 폴더 위치로 이동
cd /c/Users/yesir/study/books/DoItC

# 2. GitHub 저장소를 내 컴퓨터로 통째로 다운로드
git clone https://github.com/유저이름/저장소이름.git

# 3. 다운로드 완료 후 생성된 프로젝트 폴더 안으로 반드시 진입
cd 저장소이름

```

#### 2단계: 파일 수정 및 폴더 정리 (작업 수행)

* 파일 탐색기나 VS Code 등을 열어 소스 코드를 수정합니다.
* 폴더나 파일 이름을 바꿀 때는 터미널에서 `git mv`를 쓰거나 파일 탐색기에서 직접 변경합니다.

#### 3단계: 변경 상태 확인 및 장바구니 담기

내가 무엇을 고쳤는지 확인하고, 저장할 준비를 합니다.

```bash
# 1. 한글 경로가 깨지지 않도록 설정 (컴퓨터당 최초 1회만 해두면 됨)
git config --global core.quotepath false

# 2. 현재 어떤 파일이 수정되거나 이름이 바뀌었는지 상태 확인
git status

# 3. 변경된 모든 내용을 장바구니(Stage)에 담기
git add .

```

#### 4단계: 내 컴퓨터에 버전 기록 남기 (Commit)

장바구니에 담긴 변경 사항을 하나의 묶음(버전)으로 로컬 저장소에 영구 기록합니다.

```bash
# -m 뒤의 큰따옴표 안에 어떤 작업을 했는지 명확히 메모를 남깁니다.
git commit -m "Fix: 수정 완료 및 폴더 이름 정리"

```

#### 5단계: GitHub 온라인 저장소에 최종 업로드 (Push)

내 컴퓨터에 기록된 최신 버전을 인터넷(GitHub) 공간으로 밀어 올립니다.

```bash
# 기본 브랜치(main)로 변경 내역을 업로드합니다.
git push origin main

```

### 💡 자주 쓰는 필수 명령어 요약 리스트

| 명령어 | 역할 | 비고 |
| --- | --- | --- |
| `pwd` | 현재 내가 서 있는 폴더 경로 확인 |  |
| `ls -al` | 현재 폴더에 있는 모든 파일/폴더 목록 보기 |  |
| `git status` | 상태 확인 (수정된 파일, 장바구니 담긴 파일 표시) | 가장 자주 씀 |
| `git log` | 지금까지 내가 남긴 Commit 기록(역사) 보기 |  |
| `git remote -v` | 현재 이 폴더가 어떤 GitHub 주소와 연결되어 있는지 확인 |  |

나중에 새로운 리포지토리를 관리하거나 기존 코드를 수정할 때, 이 가이드를 위에서부터 아래로 한 줄씩 읽으면서 명령어를 입력하시면 절대 꼬이지 않고 안전하게 관리하실 수 있습니다.
